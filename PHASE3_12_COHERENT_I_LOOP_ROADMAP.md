# PHASE3_12_COHERENT_I_LOOP_ROADMAP.md

## Purpose

Define the Phase 3.12 route from the completed Phase 3.11 behavior test into a measurement-first Coherent I-Loop campaign.

Phase 3.12 does not start by claiming consciousness or adding a semantic layer. It starts by testing whether ToyI can already show bounded coherent, pattern-recognizing, persistent growth through the existing safe operator path.

## Phase 3.12 thesis

ToyI's safest next route is:

```text
bounded stream evidence
-> recurrence/pattern observatory
-> explicit promotion
-> invariant-preserving tick
-> persistent growth measurement
-> read-only coherence monitor
-> operational self-model behind review gate
-> optional model-backed proposal layer, OBSERVED/manual only
```

## Operational vocabulary

```text
coherent:
  invariants green, transitions explainable, state reports match actual state

pattern-recognizing:
  recurring structures recognized as bounded structural evidence

growing:
  persistent developmental evidence increases through accepted constructors and saturates under repetition

I:
  operational self-model anchored to COGITO_ID and repo-local constraints, not subjective experience
```

## Phases

### Phase 3.12a — Mission sync and synthesis

Install the mission, define the operational target, and record non-goals.

### Phase 3.12b — Existing-route observatory

Use existing commands only:

```text
/stream
/stream-summary
/stream-candidates
/stream-promote
/step
/state
/save-session
/load-session
```

Measure repetition, alternation, novelty, contradiction pressure, saturation, and cold-start continuity.

### Phase 3.12c — Pattern Ledger design

Design a bounded persistent ledger for recurring structural evidence. It remains below truth, agency, PtCns, and semantic interpretation.

### Phase 3.12d — Coherence Monitor design

Design a read-only coherence snapshot over state, stream/pattern evidence, persistence, and invariant health.

### Phase 3.12e — SelfModel / Growth Ledger roadmap

Design an operational self-model and growth ledger, but do not implement without explicit review-gate acceptance.
